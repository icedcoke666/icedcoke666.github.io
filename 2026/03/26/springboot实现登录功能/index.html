今日知识点：Springboot中如何实现简易的登录功能

一句话总结关系（最核心）

  1. JwtProperties：存配置（密钥、过期时间）
  2. JwtUtil：生成、解析 token 的工具
  3. JwtTokenInterceptor：拦截器——检查每次请求有没有 token
  4. WebMvcConfiguration：把拦截器注册到 SpringBoot 里

它们一起干一件事：
登录 → 生成 token → 以后每次请求都验证 token → 实现登录状态

具体实现逻辑

1.Jwtproperties类：JWT的配置类，用于配置JWT的参数，如过期时间、密钥等

— 管理端员工相关配置（ admin 开头）

  • adminSecretKey ：管理端 JWT 签名用的密钥
  • 只有持有正确密钥的服务端才能生成和验证令牌，防止伪造。
  • adminTtl ：管理端 Token 的过期时间（Time To Live）
  • 单位通常是毫秒，比如 7200000 代表 2 小时，过期后需要重新登录。
  • adminTokenName ：管理端请求头里 Token 的字段名
  • 比如 token 或 Authorization ，前端在请求头里要带这个字段名才能被后端识别。

—用户端微信用户相关配置（ user 开头）

  • userSecretKey ：用户端 JWT 签名用的密钥
  • 和管理端密钥分开，避免两端令牌互相伪造，更安全。
  • userTtl ：用户端 Token 的过期时间
  • 通常比管理端更长（比如 7 天），因为用户端不需要频繁登录。
  • userTokenName ：用户端请求头里 Token 的字段名
  • 可以和管理端一样（比如都叫 token ），也可以区分开（比如 user_token ）。

💡 为什么要分两套？

  • 角色隔离：管理端（员工）和用户端（微信用户）是完全不同的角色，不能用同一个 Token 登录。
  • 安全隔离：两套不同的密钥，即使一端泄露，另一端也不受影响。
  • 体验优化：用户端 Token 过期时间更长，不用频繁登录；管理端过期时间短，更安全。

2.JwtUtil（工具类）：生成 token、解析 token

登录时用它生成 token 返回给前端。（如果登录失败就不会调用JwtUtil生成 token）

3.JwtTokenInterceptor（拦截器）：每次请求都检查 token

流程：

  1. 前端发请求，请求头带 token
  2. 进入拦截器
  3. 获取 token
  4. 用 JwtUtil 解析
  5. 解析成功 → 放行
  6. 失败 → 不让访问（未登录）

4. WebMvcConfiguration（配置类）：把拦截器注册到 Spring，让它生效

管理端拦截器

  •  addInterceptor(jwtTokenAdminInterceptor) ：将管理员拦截器加入注册器。
  •  addPathPatterns(“/admin/**”) ：拦截所有以  /admin/  开头的请求（比如  /admin/order/list 、 /admin/goods/add  等）。
  •  excludePathPatterns(“/admin/employee/login”) ：排除登录接口，让管理员可以直接访问登录页，无需先携带 Token。

用户端拦截器

  •  addInterceptor(jwtTokenUserInterceptor) ：将用户拦截器加入注册器。
  •  addPathPatterns(“/user/**”) ：拦截所有以  /user/  开头的请求（比如  /user/order/submit 、 /user/address/list  等）。
  •  excludePathPatterns(…) ：排除两个接口：
  •  /user/user/login ：用户登录接口，允许未登录访问。
  •  /user/shop/status ：店铺营业状态接口，公开可访问。

🔥 整套登录流程（最关键，背会）

第一步：登录接口

  1. 用户输账号密码
  2. 查数据库
  3. 正确 → JwtUtil 生成 token
  4. 返回 token 给前端

第二步：前端保存 token

放到请求头里，每次访问都带上

第三步：访问其他接口（如 /user/list）

  1. 请求进入后端
  2. JwtTokenInterceptor 拦截
  3. 拿到 token
  4. JwtUtil 解析
  5. 解析成功 → 放行
  6. 接口可以拿到当前用户ID

登录逻辑

controller层，传入DTO对象，调用service层，若service层校验成功，则获取其返回的employee对象，生成jwt令牌，将employee对象封装成VO对象返回给前端

service层，调用mapper层，根据用户名查询数据库中的数据，若查询结果为空，则抛出用户不存在的异常，若结果不为空，则对密码进行md5加密，再进行比对，若密码一致，则返回该员工对象，若密码不一致，则抛出密码错误的异常，若查询结果不为空，则返回该员工对象

mapper层，根据用户名查询数据库中的数据，返回该员工对象
